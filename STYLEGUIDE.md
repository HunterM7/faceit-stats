# Styleguide

Этот файл фиксирует базовые правила оформления фронтенд-кода в проекте.

## Компоненты и SCSS

- SCSS оформляется только по БЭМ (`block`, `block__element`, `block--modifier`).
- Компонент и его стили лежат в одной директории.
- Директория компонента, файл компонента и файл стилей имеют одинаковое имя (в kebab-case).
- Имена SCSS-классов пишутся в kebab-case.
- Для условного формирования CSS-классов используется функция `classNames` из `@/utils/classNames`.
- Запрещена деструктуризация `props` в блоке параметров функции компонента.
- `props` деструктурируются только первой строкой в теле компонента.
- Деструктуризация `props` пишется в одну строку, пока не достигнут лимит длины строки.
- Пропсы компонента типизируются через `PropsWithChildren<T>` (если компонент принимает `children`).
- Если компонент не должен принимать `children`, используется тип без `PropsWithChildren`.

## Экспорты компонентов

- Для компонентов запрещены `default export`.
- Используются только именованные экспорты (`export function ComponentName() {}` / `export const ...`).
